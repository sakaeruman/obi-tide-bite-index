# OBI - Okikamuro Bite Index 🎣

山口県の漁師が Claude Code で1日で作った、地元特化の食い時合指数。

**Live Demo:** https://sakaeruman.github.io/obi-tide-bite-index/
（毎朝 5:00 JST に GitHub Actions で自動更新）

---

## これは何

OBI（Okikamuro Bite Index）は、山口県周防大島町**沖家室島**周辺・**伊予灘**水域に特化した、一本釣り向けの「食い時合指数」です。

タイドグラフBI（Bite Index）の汎用指数は全国の平均値で出されますが、沖家室の地形・潮流・水温は周防大島本島とすら違います。「自分の漁場の数値」を持ちたい、というのが出発点。

## なぜ作ったか

仲買ネットワークの目利きには勝てない。長年の勘にも勝てない。
でも AI なら、勘を**数式化して残せる**領域がある。

- 過去の釣行ログを再現できる説明変数を、自分で1個ずつ積み上げたい
- 地元特化の指数を、毎朝スマホで5秒で確認したい
- 後輩の漁師にも引き継げる形にしたい

タイドグラフBI に文句があるわけではなく、**自分の海の数式は自分で書く**しかない、というだけの話。

## OBI 数式

```
OBI(t) = [ w2·f_dh + w3·U + w4·B_twilight + w5·M_moon + w6·S_season ]
         × P_temp × P_pressure
```

各項の根拠：

- **f_dh** … 潮位変化率 dh/dt の正規化。沖家室は満干差より「動いているか」が効くため主軸に。
- **U** … 流速の代理値。v1 は dh/dt から推定、v2 で MSIL（瀬戸内海流動モデル）統合予定。
- **B_twilight** … 薄明帯（マズメ）ボーナス。Skyfield で日出没・市民薄明を秒単位で算出。
- **M_moon** … 月齢補正。大潮・小潮の周期と新月・満月夜の活性を分離して係数化。
- **S_season** … 季節係数。8魚種それぞれの旬・回遊と漁期カレンダー v1 を反映。
- **P_temp** … 水温ペナルティ。魚種ごとの適水温帯から外れると指数を抑制。
- **P_pressure** … 気圧変化ペナルティ。低気圧通過直後の食い渋り対応。

v1 の重み（w2〜w6）は仮置き。v2 で**自分の釣行ログをベイズ更新**して校正していく。

## 作る過程で見つけた知見（一番の見せ場）

AI を「書記係」ではなく「研究員」として使うと、現場の俗説をひっくり返せる場面が出てくる。OBI 構築のために 8 魚種をリサーチした過程で見つけた発見：

### マダイは赤を見ていない
**LWS opsin pseudogene**（Wang et al. 2009）。マダイは赤を感知する LWS オプシン遺伝子が偽遺伝子化しており、赤色光の感度は他色より明確に低い。
**「マダイは赤に反応する」は俗説寄り**。実際は形状・動き・コントラストで反応している可能性が高い。赤色ルアーが釣れる理由は別にある。

### サワラの「下から襲う説」は俗説寄り
サワラは浮袋を持たない（physoclistous ではなく非保持型）。縦方向の急速移動はエネルギーコストが高く、生理的に「下から急襲」の頻度は低いと考えるほうが整合的。横方向の高速追尾が主戦法。

### サバの遊泳速度 ≒ 18 体長/秒
Wardle (1988) のキネマティクス研究。シマノ公式のジギング推奨アクション速度は、この学術値に**ちゃんと合っている**。メーカーの説明文は雰囲気で書かれているように見えて、裏取りされているケースがあった。

### 歯魚対策の業界正解＝ハリス上部 20-30cm だけ複合メタル化
タチウオ・サワラ・サゴシ対策。全長ワイヤーは食い渋り、フロロ単体は歯切れ。
**オーナー F-6330** など、ハリス上部だけ複合メタルにする製品が既に存在する。情報が漁師界隈に降りていないだけだった。

---

これらは「漁師の経験 + AI のリサーチ能力」の掛け算でしか出てこない発見。
**俗説を疑うときに AI は本当に役立つ**、というのが OBI を作って一番得た学び。

## AI をどう使ったか（Claude Code 使用例として）

1. **Ultracode ワークフロー × 4 本の並列実行**
   1 つの仮説に対して、独立したエージェントを 4 本同時に走らせる。

2. **3 軸補完リサーチ**
   - 軸 A: 学術論文（Google Scholar、Wang 2009 / Wardle 1988 等）
   - 軸 B: メーカー資料（シマノ・ダイワ・オーナーの公式説明）
   - 軸 C: 製品スペック（実在する SKU、価格、入手性）

   3 軸が一致したものだけを「採用」、ズレたものは「俗説候補」として記録。

3. **並列エージェントで俗説を見抜く**
   1 本目が「マダイは赤に反応する」と書いてきても、3 本目が LWS pseudogene の論文を引いてきて矛盾を検出 → 採用基準で却下、という流れが起きる。

4. **AI は書記係ではなく研究員**
   要約させるのではなく、**自分が知らない領域の論文・特許・製品仕様を集めて突合させる**使い方が刺さる。

## セットアップ（ローカル）

```bash
git clone https://github.com/sakaeruman/obi-tide-bite-index.git
cd obi-tide-bite-index

python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt

python3 -m src.daily
```

`docs/index.html` が更新されればローカル実行成功。`open docs/index.html` で確認できます。

## 他地点への流用

`config.yaml` の以下を書き換えれば、別の漁場でも動きます：

```yaml
location:
  name: "沖家室"
  lat: 33.86
  lon: 132.27

tide_station: "TS"   # 気象庁の徳山潮位コード
weather_point: "..."  # JMA の地点コード
species: [マダイ, サワラ, タチウオ, ...]
```

地形・潮流が違えば数式の重みも変わるはず。**フォークして自分の海の OBI を作ってください**。
